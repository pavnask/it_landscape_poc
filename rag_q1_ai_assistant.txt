Фрагмент текущего enterprise-ландшафта:

1) В компании уже есть AI-направление:
- АС ИИ Хаб Юпитер: jupiter.app_system.aihub
- Класс: мультиагентная система
- Есть подсистема модели DeepSeek: jupiter.app_system.aihub_model_ds1
- Жизненный цикл: в разработке / приобретение

2) В контуре новостей уже есть AI-паттерн:
- АС ECM Новости имеет интеграцию с AI-сервисом сводки новостей
- Используется jupiter.srv.ai_news_summary
- Есть внешняя система "ИИ Агенты создания новостей"

3) Во фронтальных каналах уже есть мобильное приложение:
- jupiter.app_system.mp_favorite_client
- Реализует функцию личного кабинета клиента
- Есть интеграция с новостным бэком

4) Вывод по паттернам:
- AI-функциональность уже выделяется в отдельные системы / сервисы
- Во фронтальные каналы AI лучше встраивать через отдельный сервисный или API-слой
- Не выглядит правильным встраивать AI-логику напрямую в мобильный фронт как жёстко связанную логику
