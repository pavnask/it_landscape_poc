3

Почему RAG и Skills не решили задачу

RAG

Что делает хорошо

  • Даёт факты о ландшафте
  • Повышает релевантность ответа
  • Уменьшает галлюцинации

Что не делает

  • Не навязывает правильный паттерн
  • Не создаёт enterprise bias
  • Не заставляет модель выбрать решение
Skills.md

Что ожидали

  • “Запишем всё в skills”
  • “Модель будет отвечать как архитектор”
  • “Rules заменят обучение”

Что получилось

  • Модель буквально следует правилам
  • Теряет нюансы контекста
  • Выбирает самое простое совпавшее правило
  • Даёт shallow или смещённые рекомендации
Skills — это инструкция, а не мышление. Нельзя превратить модель в архитектора, просто записав правила.
4

Что реально делает LoRA

Поведение Base 7B 7B + LoRA
Детектит anti-pattern ❌ непоследовательно ✅ последовательно
Выбирает integration pattern ❌ расплывчато ✅ точно
Уровень уверенности ❌ низкий ✅ высокий
Enterprise bias ❌ слабый ✅ сильный
Пример победившего ответа LoRA: “File exchange как основной online-механизм — anti-pattern; нужен API через gateway и integration layer, при необходимости с async transport.”
LoRA не добавляет новые знания. LoRA меняет то, как модель принимает решение внутри уже имеющихся знаний.
5

Технический прорыв: 7B LoRA на commodity hardware

Что мы доказали
  • 7B модель реально работает на Mac через MLX
  • LoRA training реально запускается локально
  • Inference с адаптером быстрый и воспроизводимый
  • Нужен не GPU-кластер, а грамотный стек
Почему это важно
  • Быстрые итерации без дорогой инфраструктуры
  • Низкий порог входа для команды
  • Enterprise reasoning можно дообучать локально
  • Это уже не лабораторная игрушка, а практический путь
Мы научились делать LoRA на большой модели на обычном Mac. Это переводит fine-tuning из “дорогой R&D” в “рабочий инженерный инструмент”.
6

Главные выводы и ответы скептикам

Вопрос / тезис Ответ
“Может, хватит только RAG?” Нет. RAG даёт факты, но не делает архитектурный выбор
“Может, достаточно skills.md?” Нет. Skills делают модель послушной, но не создают reasoning
“Может, просто взять 7B?” Нет. Большая модель умнее, но без LoRA остаётся нерешительной
“LoRA просто пользуется знаниями базы?” База даёт capacity. LoRA меняет decision bias и default architectural choices
“Можно ли всё записать в skills и закрыть проблему архитектора?” Нет. Skills — это operational playbook, а не источник архитектурного мышления
Base model — знает. RAG — информирует. Skills — направляют. LoRA — решает.
