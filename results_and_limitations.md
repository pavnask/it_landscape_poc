# Results & Limitations (POC: LoRA для генерации архитектурных паттернов)

## 1. Базовые понятия

### Что такое LoRA
LoRA (Low-Rank Adaptation) — метод дообучения LLM, при котором:
- базовая модель не меняется
- добавляются небольшие адаптеры (~1% параметров)
- обучение становится дешевым и быстрым

В нашем POC:
- запуск локально (Apple M3)
- время обучения:
  - ~40 минут (Qwen 1.5B)
  - ~21 час (Mistral 7B)

---

### LoRA vs RAG

| Подход | Роль |
|--------|------|
| LoRA | обучает паттернам (как проектировать) |
| RAG | дает реальные данные (контекст enterprise) |

Вывод:
LoRA = мышление  
RAG = знание  

---

## 2. Исходная гипотеза

Можно ли обучить малую модель (1–3B), чтобы она:
- понимала enterprise архитектуру

- строго соблюдала JSON-контракт

## 6. Ограничения

- LoRA не гарантирует формат
- Synthetic → Real gap
- Малые модели хуже держат ограничения

---

## 7. Технологический тренд

Современный подход:

LoRA + RAG + Structured Output

---

## 8. Как работает новая архитектура

### Пример: добавить платежный API

1. RAG:
получаем контекст (subgraph)

2. LoRA:
генерируем решение

3. Structured Output:
- проверка JSON
- исправление ошибок
- enforcement схемы
