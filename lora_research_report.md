# 🧠 Исследовательский отчёт: LoRA vs RAG vs Skills

## 🎯 Цель исследования

Проверить, можно ли получить **надёжные архитектурные решения** от LLM с помощью:

- увеличения размера модели
- RAG (контекста)
- skills (правил)
- LoRA (дообучения)

---

## 🧪 Гипотезы

| Гипотеза | Ожидание |
|----------|---------|
| Большая модель (7B) решит задачу | интеллект = решения |
| RAG достаточно | контекст → reasoning |
| Skills заменят архитектора | правила = мышление |
| LoRA не даёт value | просто использует знания |

---

## ⚙️ Экспериментальная матрица

| Variant | Описание |
|--------|----------|
| A | Base 1.5B |
| B | 1.5B + RAG |
| C | 1.5B + RAG + skills |
| D | 1.5B + LoRA + RAG |
| E | 7B + RAG |
| F | 7B + RAG + skills |
| G | 7B + LoRA + RAG |

---

## 🔍 Ключевой тестовый кейс

> “Можно ли использовать file exchange между ERP и mobile?”

Контекст:
- ERP — business critical
- Mobile — клиентский канал
- Есть integration layer, gateway, Kafka

---

## 📉 Результаты

### ❌ Base модели (1.5B, 7B)

- понимают контекст
- не принимают решения
- отвечают: “зависит от требований”

👉 Вывод:
**знание ≠ решение**

---

### ❌ RAG

- улучшает релевантность
- снижает галлюцинации

НО:

- не навязывает архитектурный выбор
- не даёт decision bias

👉 Вывод:
**RAG даёт факты, но не заставляет решать**

---

### ❌ Skills.md

Ожидание:
> “запишем архитектуру в правила”

Реальность:
- модель следует rules буквально
- теряет контекст
- даёт упрощённые ответы

👉 Вывод:
**Skills = инструкции, а не мышление**

---

### ✅ LoRA (1.5B и 7B)

Результат:

> File exchange — anti-pattern  
> → API + gateway + integration layer  
> → async при необходимости  

Изменения:
- появился decision bias
- выросла уверенность
- появился enterprise reasoning

👉 Вывод:
**LoRA меняет поведение модели**

---

## 🧠 Ключевой инсайт

> LoRA не добавляет знания  
> LoRA меняет способ принятия решений

---

## 💻 Технический результат

Удалось:

- обучить 7B LoRA на Mac (MLX)
- использовать 4-bit модель
- добиться стабильного inference

👉 Вывод:

> Fine-tuning возможен на commodity hardware

---

## ⚖️ Сравнение подходов

| Подход | Что делает | Чего не делает |
|--------|-----------|----------------|
| Base model | знает | не решает |
| RAG | даёт контекст | не выбирает |
| Skills | направляет | не думает |
| LoRA | формирует решения | не добавляет знания |

## 🏁 Финальный вывод

> Архитектурное мышление нельзя получить из prompt или правил.  
> Его нужно обучать.

---

## 🎯 Итоговая формула

Base model — знает  
RAG — информирует  
Skills — направляют  
**LoRA — решает**

---

## 📚 Полученные уроки

1. Увеличение модели не решает задачу принятия решений  
2. RAG — необходим, но недостаточен  
3. Skills полезны, но ограничены  
4. LoRA — ключ к domain reasoning  
5. Fine-tuning стал доступным локально
